# 수도권대기환경청 AI기반 우선관리 산업단지 분석시스템

측정소 데이터(PM2.5 / 오존)로 **산업단지 우선순위**를 산출하는 단일 HTML 앱입니다.
빌드·설치 없이 파일만 올리면 GitHub Pages에서 바로 동작합니다.

## 접속
- 접속코드 **1111** 입력 (아이디 없음, 코드는 화면에 표시되지 않음)

## GitHub Pages 배포 방법 (가장 쉬운 방법 · 빌드 불필요)

1. GitHub에서 새 저장소(repository)를 만든다. 예: `aqms-app`
2. 이 폴더의 파일 3개를 저장소에 업로드한다.
   - `index.html`  ← 앱 본체
   - `.nojekyll`   ← 정적 파일 정상 서빙용(빈 파일)
   - `README.md`   ← 설명(선택)
   > 웹에서 올릴 때: 저장소 → **Add file → Upload files** → 3개 드래그 → **Commit changes**
3. 저장소 상단 **Settings → Pages** 로 이동
4. **Build and deployment**
   - Source: **Deploy from a branch**
   - Branch: **main** / 폴더 **/ (root)** 선택 → **Save**
5. 1~2분 뒤 상단에 나오는 주소로 접속
   `https://<사용자이름>.github.io/<저장소이름>/`

끝입니다. 접속 후 접속코드 `1111`을 입력하면 됩니다.

## 사용법
1. 상단에서 **PM2.5 / 오존(O3)** 선택
2. **데이터 파일 업로드** (CSV 또는 XLSX)
3. **분석 대상 산업단지 선택** (검색 / 전체선택 가능)
4. **분석 실행** → 순위표 + 상위 10개 상세 보고서 + 지도

## 접속코드 변경 방법
`index.html` 을 열어 상단의 다음 줄을 찾는다:

```js
const AUTH_CODE_SHA256 = '0ffe1abd...';   // = 코드 1111 의 SHA-256
```

새 코드의 해시로 바꾸면 된다. 해시는 브라우저 개발자도구(F12) 콘솔에서:

```js
await sha256('원하는코드')   // 이 앱이 로드된 상태에서 실행 → 나온 값을 붙여넣기
```

## ⚠️ 보안 안내 (정직하게)
- GitHub Pages는 **공개 사이트**입니다. URL을 아는 사람은 페이지에 접근할 수 있습니다.
- 접속코드는 브라우저 안에서 검사하므로 **우회 가능한 "간단 접근 차단"**이지 진짜 보안이 아닙니다.
  코드(및 그 해시)도 파일 안에 들어 있습니다.
- 민감한 자료를 실제로 보호해야 한다면, 파일을 공개 저장소에 올리지 말고
  서버 인증(백엔드) 또는 호스팅 레벨 인증(Cloudflare Access, Netlify Identity 등)을 사용하세요.

## 필요 환경
- 인터넷 연결 (Tailwind · Leaflet · SheetJS를 CDN에서 로드)
- 최신 브라우저(Chrome / Edge / Safari 등)
